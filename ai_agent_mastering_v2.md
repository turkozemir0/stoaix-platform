## Bölüm 3: Teknik Mimari Derinlemesi

### 3.1 STT (Speech-to-Text) Karşılaştırması: Deepgram Nova-2 vs Flux vs Nova-3

#### Deepgram Nova-2 (Mevcut STOAIX Standart)

**Özellikler:**
- Monolingual (mono-lang per call) veya Multilingual (birden çok lang)
- **Word Error Rate (WER):** Türkçe benchmark verilmemiş, ama genel olarak %8-12
- **Latency:** ~200-300ms (real-time, streaming)
- **Fiyat:** $0.0058/min (pay-as-you-go), $0.0047/min (growth plan) [Kaynak: Deepgram pricing]

**Türkçe Destek:** Evet, multilingual modelde

**Avantajları:**
- Proven track record (2023 boyunca şirketler tarafından kullanılır)
- Cost-effective
- Türkçe desteği var

**Dezavantajları:**
- End-of-turn detection manuel (Silero VAD ile)
- Konu-spesifik kelimelere tepkisizlik (e.g., clinic procedure names)

#### Deepgram Flux (Yeni, 2024)

**Özellikler:**
- **Konuşmacı AI'ya özel tasarlanmış** (turn-taking, interruption handling built-in)
- **End-of-Turn Detection:** Integrated (no external VAD), configurable (`eot_threshold`, `eager_eot_threshold`)
- **Latency:** < 200ms (STT component)
- **Fiyat:** $0.0077/min (pay-as-you-go), $0.0065/min (growth plan) — **Nova-2 ile aynı**
- **Dil Destek:** 30+ languages including Turkish

**Türkçe Destek:** Evet, ve optimize edilmiş

**Avantajları:**
- **Turn-taking quality = humanness + latency** (600ms RTT yerine 800ms+ azaltabilir)
- Dynamic keyterm injection (data capture adımlarında "İstanbul", "Ayşe" gibi kelimeleri öğret)
- Backchanneling filtration ("mm-hmm", "evet", "tamam" = yanlış interrupt değil)
- **STOAIX'in en büyük pain point'ini çözer:** false interrupts

**Dezavantajları:**
- Yeni (maturity unknown)
- Türkçe benchmarks henüz publish edilmemiş
- LiveKit plugin support = **doğrulama gerekli**

**STOAIX Önerisi:** **Q2 2026'de A/B test başlat.** Flux enable, 10 kliniğe roll out, latency + interrupt metrics karşılaştır.

#### Deepgram Nova-3 (Gelecek Seçenek)

**Özellikler:**
- **Yeni (Ocak 2025 duyuru)**
- Nova-2'ye göre **double-digit WER reduction** (Turkish explicitly mentioned)
- 100+ dil
- Streaming + batch

**Türkçe Destek:** Açıkça optimize edilmiş, Nova-2'den %15-25 daha iyi tahmin

**Avantajları:**
- Türkçe için measurable iyileştirme
- Batch + streaming (flexibility)

**Dezavantajları:**
- Fiyatlandırması henüz bilinmiyor (Nova-2 ile aynı olacağı tahmin ediliyor)
- Flux kadar turn-taking optimized değil

**STOAIX Önerisi:** Flux'ü test et, Nova-3 çıktığında latency comparison yap.

### 3.2 LLM (Large Language Model) Seçimi: OpenAI vs Anthropic vs Groq

#### OpenAI GPT-4o Mini (Mevcut STOAIX Standard)

**Model Detayları:**
- **Latency:** ~200-300ms (API call, standard)
- **Cost:** ~$0.00015 per 1K input tokens, ~$0.0006 per 1K output tokens [Kaynak: OpenAI API pricing]
- **Context Window:** 128K tokens
- **Multilingual:** Evet, Türkçe excellent
- **Function Calling:** Native support (tool/API invocation)

**Türkçe Capability'si:** **Excellent** — clinic personas, Turkish idioms, cultural nuance

**Avantajları:**
- Türkçe sohbet quality **premium**
- Reliable production
- Ecosystem (embeddings, moderation, etc.)

**Dezavantajları:**
- Cost (longer convos = cost creep)
- Rate limiting (high volume)

**Tipik Usage:** 100 arama/gün × 3 min × 100 output tokens/min × $0.0006/1K = **~$18/gün = $540/ay**

#### Anthropic Claude (Alternatif)

**Model Detayları:**
- **Latency:** ~300-400ms
- **Cost:** ~$0.0008 per 1K input (claude-3-haiku), $0.0002 per 1K output
- **Context Window:** 200K+ (opsiyonel)
- **Multilingual:** Evet, but Turkish less battle-tested

**Türkçe Capability'si:** Good (ama OpenAI kadar native değil)

**Avantajı:** Daha ucuz, daha uzun context

**Dezavantajı:** Türkçe production reference daha az

**STOAIX önerisi:** OpenAI gpt-4o-mini kalsın (Türkçe trust'ı için)

#### Groq (Ultra-Low Latency)

**Model Detayları:**
- **Latency:** ~50-100ms (LLM inference)
- **Models:** Meta Llama 3 (open source)
- **Cost:** Per-million-token based, ~50% OpenAI'dan ucuz
- **Multilingual:** Sınırlı (Llama 3'teki Turkish support lower quality)

**Türkçe Capability'si:** **Weak** (Llama 3 English-primary)

**Avantajı:** Ultra-hızlı latency

**Dezavantajı:** Türkçe quality, API stability

**STOAIX önerisi:** Premium demoları için test et, ama production = OpenAI kalsın

### 3.4 Turn-Taking & Interruption Handling: Teknik Derinlik

**Problem:** Kötü turn-taking = **robot hissi en hızlı trigger'ıdır**

#### Silero VAD (Mevcut STOAIX)

**Nasıl çalışır:** Voice Activity Detection — ses/sessizlik binary classification

**Sorunlar:**
- "evet", "mm", "tamam" gibi backchanneling = yanlış interrupt
- Kişi cümle ortasında durup düşünürse, VAD=false, premature cut-off

**Latency:** ~100ms

#### LiveKit Adaptive Interruption Handling (Ideal Yaklaşım)

**Nasıl çalışır:**
1. VAD (Silero) + acoustic signal analysis (cut-in patterns)
2. Backchanneling filter (short utterance + low energy = not interrupt)
3. Transcript alignment (STT confidence + partial word detection)
4. LLM-based turn detection (upcoming LiveKit feature)

**Avantaj:**
- ~800ms false interrupt rate from 30% → 5%
- Türkçe backchanneling ("tamam", "haklısın") native support

**Implementasyon (LiveKit Agents):**
```python
from livekit.agents import TurnHandlingOptions
from livekit.plugins.turn_detector.multilingual import MultilingualModel

agent_session = AgentSession(
    turn_handling=TurnHandlingOptions(
        turn_detection=MultilingularModel(),  # LLM-based, Türkçe support
        mode="dynamic",
        min_delay=0.4,    # ms before responding (default 0.5)
        max_delay=2.5,    # ms max silence tolerance
    ),
    allow_interruptions=True,
    min_interruption_duration=0.5,  # must be >0.5s to interrupt
    min_interruption_words=2,        # must be 2+ words (filters "evet")
    resume_false_interruption=True,  # if VAD wrong, resume agent speech
)
```

**STOAIX Önerisi:** **Hafta 1'de implement et.** Silero VAD'ı MultilingualModel'le değiştir. Türkçe test cliniği ile A/B karşılaştır.

#### Deepgram Flux: Built-In End-of-Turn Detection

**Nasıl çalışır:**
- STT model itself detects turn-boundary (not post-hoc VAD)
- `eot_threshold`, `eager_eot_threshold`, `eot_timeout_ms` configs
- Dynamic keyterm injection (step-spesifik words)

**Avantaj:**
- Native turn detection = latency azaltma
- Turkish word boundary awareness
- Clinic-specific vocabulary biasing (e.g., "implant", "zirkonyum", "muayene")

**STOAIX Önerisi:** Flux + MultilingualModel = ideal combo. Flux STT handles turn-boundary, LLM turn-detector handles logic.

### 3.5 Prompt Mimarisi: Monolitik vs State Machine

#### Monolitik Prompt (Mevcut STOAIX)

**Yapı:**
```
# ROLE & OBJECTIVE
# PERSONALITY
# TOOLS
# RULES
# SALES
# ESCALATION
[Tüm bilgi 1 system message'da]
```

**Sorunlar:**
- LLM farklı contexts'te tutarsız davranır (greeting'de satış yapıyor, FAQ'de over-explaining)
- Instruction "conflicts" (e.g., "short" ama "detailed explain")

#### State Machine Prompt (Önerilen)

**Yapı:**
```
# ROLE
# CURRENT CONVERSATION STATE: [opening|discovery|faq|objection|booking|handoff]
# STATE-SPECIFIC RULES:
  if state == "opening":
    max_length: 2 sentences
    ask_only: reason for call
    tone: warm, brief
  if state == "discovery":
    max_length: 3 sentences
    ask_only: 1 qualifier question
    rule: answer first, ask later
  if state == "booking":
    max_length: 1 instruction at a time
    ask_only: confirmation
# ESCALATION RULES
```

**Avantajı:**
- Consistent behavior per state
- Easy to debug ("agent was in FAQ state, why did it sell?")
- Türkçe clinic personas (Elif) per-state optimize

**Implementasyon (LiveKit):**
```python
class ClinicAgent(Agent):
    def __init__(self):
        super().__init__(instructions=SYSTEM_PROMPT)
        self.conversation_state = "opening"  # Track state
        self.state_rules = {
            "opening": {"max_sentences": 2, "ask_type": "reason"},
            "discovery": {"max_sentences": 3, "ask_type": "qualifier"},
            "faq": {"max_sentences": 2, "ask_type": "detail"},
            "booking": {"max_sentences": 1, "ask_type": "confirmation"},
        }
    
    async def on_user_turn_completed(self, turn_ctx, user_message):
        # Determine next state based on user message
        intent = self.detect_intent(user_message)  # booking? objection? faq?
        self.conversation_state = self.get_next_state(intent)
        
        # Inject state rules into system prompt
        state_instruction = self.format_state_rules(self.conversation_state)
        updated_prompt = SYSTEM_PROMPT + "\n" + state_instruction
        
        # Continue with updated prompt
        ...
```

# Few-shot examples for clinic sales state

example_1_discovery:
  user: "Saç ekimi hakkında bilgi istiyorum."

  why_good: |
    - Opening statement (Harika)
    - Clear value (nasıl yardımcı)
    - One question only
    - ~40 words (not too long)

example_2_objection:
  user: "Çok pahalı gibi."
  agent_response: |
    "Anlıyorum. Bir sorum var: 
    sizin için saç ekiminin başarısı ne yaşamanız demektir? 
    [wait for answer] 
    Bunu duyunca, fiyat daha anlamlı hale gelir."
  why_good: |
    - Acknowledge (Anlıyorum)
    - Reframe (value-first, not price defense)
    - One question (dialogue, not monologue)

example_3_faq:
  user: "Kaç dakikada biter?"
  agent_response: |
    "Günü gelen cerrahi 6-8 saat sürüyor. 
    Ama endişelenme, bölümlenmiş — kahvaltı breaks vs. var. 
    Halen 8 saatini kalmak uygun mu?"
  why_good: |
    - Direct answer (no preamble)
    - Reassurance (Ama endişelenme)
    - One follow-up question (not "Have more questions?")

example_4_wrong:
  user: "Fiyat nedir?"
  agent_response_WRONG: |
    "Merhaba! Saç ekimi tedavi seçeneklerimiz var. 
    Fiyatlandırmamız işlem türüne göre değişiklik gösterir. 
    Seninle birlikte en uygun paketi bulabiliriz. 
    Lütfen name, phone, city paylaş mı?"
  why_bad: |
    - Too long (3 sentences → should be 1-2)
    - Doesn't answer directly ("Fiyat nedir" → no number given)
    - Asks too many things (name, phone, city)
    - Feels like form, not human conversation

### 5.3 Common Pitfalls & Fixes

| Pitfall | Wrong | Correct |
|---|---|---|
| **Too long** | "Kliniğimiz saç ekimi, dental implant, estetik cerrahisi ve çeşitli medikal estetik uygulamaları sunmaktadır. Bunların her biri için süreç farklılık göstermekte olup ön değerlendirme sonrasında net bilgi paylaşılmaktadır." | "Saç ekimi, diş implant, estetik cerrahi yapıyoruz. Sende hangisine ilginiz var?" |
| **Asks before answering** | "Pardon, siz saç ekimi mi, diş mi, estetik mi arıyorsunuz?" | "Saç ekimi mi soruyordunuz?" [wait] "Evet, saç ekimi oldukça popüler, detay vereyim mi?" |
| **Repetitive phrasing** | agent: "Tabii ki" (3x aynı call'da) | agent: 1x "Tabii", 1x "Memnuniyetle", 1x "Elbette" |
| **Bot language** | "Teşekkür ederim sorunuz için. Bunun için bize başvurduğunuz için minnettar." | "Çok iyi soru." |
| **Lists** | "1. Saç ekimi 2. Diş implant 3. Estetik" | "Üç seçeneğiniz var: saç ekimi, diş implant, ya da estetik. Hangisi daha çok sizi merak ettiriyor?" |
| **Fake enthusiasm** | "Harika soru! Çok mutluyum bunu cevaplama fırsatı için!" | [Just answer directly, naturally] |
| **Numbers in digits** | "2000 dolar" | "iki bin dolar" |
| **Wrong confirmation** | [After user says "0532 123 45 67"] agent: "Tamam, kaydettim" | agent: "Numarayı doğru aldım mı: sıfır beş otuz iki, yüz yirmi üç, kırk beş, altmış yedi. Doğru mu?" |
